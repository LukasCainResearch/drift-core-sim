1. The Concept: Non-Local Dynamics

Standard cryptographic algorithms (AES, SHA) are "Local." Their behavior depends on fixed block sizes (e.g., 128 bits) and localized bitwise operations.

Drift Systems introduces Non-Local Maps. By checking the residue of a number modulo a prime $P$ (e.g., $n \pmod 3$), we force a dependency on the entire bit string.

A single bit flip at the LSB (Least Significant Bit) ripples all the way to the MSB (Most Significant Bit) to determine the next arithmetic step. This creates maximal diffusion instantly.

Logic Flow:

1. State Register $S$
  ↓
2. Global Check: $S \pmod 3$
  ↓
3. Feedback: Select Multiplier $q$
  ↓
4. Arithmetic Op: $S' = qS + d$
