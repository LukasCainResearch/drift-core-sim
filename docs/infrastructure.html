1. Overview

What it solves:

  • Stale/late work acceptance in distributed job and event systems
  • Replay of valid work items (idempotency failures, duplicate commits)
  • Out-of-order delivery and sequence drift across producers
  • Partition and partial validator failure with safe degraded-mode policy

How to read the demos: they demonstrate invariants (freshness window, replay cache, monotonic ordering, quorum), using tutorial math — not the proprietary construction.

DECISION INVARIANTS

Gate enforces: max-age freshness + deadline budget + replay cache + monotonic seq → ACCEPT/REJECT/TIMEOUT. Validators enforce: 2-of-3 quorum → COMMIT or degraded-mode.

FAILURE CLASSIFICATION

REPLAY vs STALE vs ORDER vs TIMEOUT are treated differently. This enables policy-driven operations: reject unsafe work, continue in degraded-mode, and recover once quorum returns.
